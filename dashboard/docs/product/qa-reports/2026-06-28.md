## Check 2 — Architecture conformance (code audit)

**AC9 — Electron security: PASS**  
`main/index.ts`: `contextIsolation: true`, `nodeIntegration: false`, `sandbox: false`. Preload bridge exposes only the allow-listed `window.api`. ✓

**AC13 — git via `execFile` + arg arrays: PASS**  
`git-service.ts` uses `child_process.execFile` with `['status', '--porcelain']` and `['rev-list', '@{u}..HEAD']` as arg arrays. No shell string interpolation anywhere. ✓

**AC14 — Tolerant parsing: PASS**  
Both `registry-reader` and `state-reader` ignore unknown fields, default missing optionals, and emit schema-version warnings without crashing. Tested by 12 targeted unit tests. ✓

**AC10 — Cap-6 concurrency: FAIL (significant)**  
`p-limit` is installed as a dependency but **never imported or used** in `git-service.ts`. The production `gitService` singleton wraps raw `execFile` with no concurrency limit. `App.tsx` fires the background batch with `entries.forEach((entry) => loadGitState(...))` — unbounded fan-out. With 100 repos this spawns 200 child processes simultaneously, violating architecture ADR and potentially overwhelming the system.

**AC3 / AC11 — SWR revalidation on project open: FAIL (significant)**  
`App.tsx` defines `handleSelectProject` (which calls `addRecent` + triggers git revalidation) but **never passes it to `<Sidebar />`**. Sidebar calls `selectProject` directly from the Zustand store, which only updates `selectedPath`. Neither `window.api.addRecent` (IPC persistence) nor `loadGitState` (SWR revalidation) fires on click. Two contracts broken:
- Recently-opened list doesn't persist between restarts.
- Git state doesn't revalidate when a project is opened (architecture §4: "git follows the SWR pattern — cached value rendered instantly… a background revalidate runs").

## Check 4 — Non-functional

**AC12 — Offline-first: FAIL (significant)**  
`index.html` loads Space Grotesk from `https://fonts.googleapis.com` / `https://fonts.gstatic.com`. This is a live CDN request. When offline, the font fails to load and the browser falls back to `system-ui` — the distinctive industrial typography of the design concept is lost. Architecture driver #3 is "Offline-first & local-only — no network." Space Grotesk must be bundled.

**AC9 — CSP: PASS (conditionally)**  
HTML CSP allows `fonts.googleapis.com` and `fonts.gstatic.com`. Once the font is bundled (fixing above), these directives should be removed to tighten the CSP.

**AC18 — Keyboard navigation: PASS**  
Sidebar items are `<button>` elements; all interactive elements have focus-visible ring (`:focus-visible { outline: 2px solid var(--accent-hi) }`). Tab order follows DOM order.

**AC18 — WCAG AA contrast (visual inspection required)**

## Check 5 — Minor issues

**M1 — Watcher leak on repeated `list-projects` calls (minor)**  
`handlers.ts:startWatching()` calls `watcher.start()` each time `list-projects` is invoked. `WatcherService.start()` creates a new `chokidar.watch()` instance without closing the previous one (`this.watcher` is overwritten, old watcher keeps running). The `onRegistryChange` handler in App.tsx calls `window.api.listProjects()` — so every registry change triggers a leaked watcher. Over time this accumulates file descriptor usage.

**M2 — Feature back-navigation passes wrong identifier (minor)**  
`App.tsx:152`: `onBack={() => selectProject(selectedEntry.parentId ?? null)}`. `parentId` is a registry entry ID string (e.g. `"agent-c-dashboard"`), not a path. `selectProject` matches by `p.path`, so the parent is never found. No features exist in v1 yet, so this is dormant, but will fail when features are registered.

**M3 — Footer shows render time, not sync time (minor)**  
`Footer.tsx`: `formatSyncTime()` calls `new Date().toLocaleTimeString()` at render, so "Last synced" always shows the moment the footer renders, not when the actual sync completed. The UX doc specifies "Last synced 2 minutes ago" style feedback. A `lastSyncedAt` timestamp should be stored in the Zustand store and updated on each `setSyncStatus('idle')` call.
